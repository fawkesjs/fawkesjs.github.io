FawkesJs

A NodeJs framework build on top of Express, TypeScript and Sequelize, with strict swagger param checking and ACL middleware support

Why FawkesJs?

FawkesJs is inspired by Loopback and Laravel. We want to make programmers' life easier by including modular design of Laravel and build in swagger support of Loopback.

Optional Type Safety

We use Typescript as our main language and use a lot of Interface to guide you. It's recommended to use along with Atom as your editor.

Express + Swagger

It has built in swagger document generation. All the api endpoint has the type/strlen verification according to your Swagger 2.0, from GET query to POST body.

Model View Controller

Model–view–controller (MVC) is a software architectural pattern for implementing user interfaces on computers. It has been adopted by many web framework.

Built in ORM

Although we already have Model, we still want our life easier by implementing Object-relational mapping (ORM). Useful when create or update database records.

Middlewares

Optional middlewares such as AcessToken Middleware, ACL middleware and RestParam Middleware. You can also easily create your own middleware with Promise.

Organised Route

Organize api endpoint using folders structure. All the required parameters is set together with the route. Each route can be protected by ACL easily.

Features

  • Swagger Generation
  • Configure middlewares in an array
  • FawkesJs-starter with User, ACL ecosystem
  • Swagger Param Validation on Api Endpoint
  • Organised Route in folders structure
  • Model View Controller (MVC)
  • ORM support

Get Started


git clone https://github.com/fawkesjs/fawkesjs-starter
npm install && npm run start

  - config

    - config.{env}.ts
    - datasource.ts
    - datasource.{env}.ts
    - swagger.ts
    - swagger.{env}.ts
  - controller
  - middleware
    - index.ts
  - model
  - module
  - orm
    - index.ts
  - route
    - index.ts
    - api
      - index.ts
      - account
        - index.ts
